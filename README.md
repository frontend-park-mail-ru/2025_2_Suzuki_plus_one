- Репозиторий команды: `Сузуки + 1`
- Проект: `Netflix`
- Деплой: https://popfilms.ru/

## Основные пункты разработки проекта

- В ветку main нельзя напрямую коммитить
- Всю разработку ведем в ветке dev, отведенной от main
- Для каждой задачи от dev отводится отдельная ветка с названием вида <TIKET_ID>-<feature-name>, где <TIKET_ID> - id тикета в трекере, <feature-name> - краткое название выполняемой фичи/задачи
- По завершении разработки задачи создается Pull Request в dev, по которому я или ваши сокомандники проводят ревью. Название Pull Request'а - строка вида <TIKET_ID>: <description>, где <TIKET_ID> - id тикета в трекере, <description> - краткое описание выполненной работы. Более подробно выполненную работу можно описать в соответствующем поле.
- Перед каждым РК ветка dev вливается в main.
